0. 프로젝트 정체성
================================================================

kFarmAI는 단순 농업 블로그, 농자재 쇼핑몰, 업체 디렉토리 사이트가 아니다.

핵심 방향:
“농업·식물 문제 검색 → 공식기관 출처 기반 답변 → AI 진단 → 주변 농약사/농자재 연결 → 업체 광고/유료등록/데이터 리포트 수익화”

현재 개발자는 전문 개발자가 아니며, AI 협업 기반의 바이브코딩 방식으로 개발 중이다. 따라서 모든 작업은 유지보수 가능성, 파일 구조 안정성, 백업, 로그, 문서화, 보안, 회사 자산과 개인 프로젝트 분리를 우선한다.

C:\Users\user\PERSONAL\dev\kfarmai\02_dev\frontend

1. 장비 및 자산 분리 원칙
================================================================

kFarmAI는 개인 프로젝트이다.

장비 구분:
A = 개인 개발 PC
D = 개인 핸드폰
B = 회사 PC
C = 회사 핸드폰

역할:
A 개인 PC:
- 개발, 코드 수정, 테스트, SEO worker 실행, Git 관리, 문서화, OpenClaw 설치/운영 가능

D 개인 핸드폰:
- 모바일 화면 확인, 원격 상태 확인, 비상중지
- 코드 수정 금지

B 회사 PC:
- kFarmAI 개인 프로젝트 사용 금지

C 회사 핸드폰:
- kFarmAI 개인 프로젝트 사용 금지

금지:
1. B/C에 kFarmAI 코드 저장 금지
2. B/C에 .env, API 키, 토큰 저장 금지
3. B/C에서 kFarmAI GitHub 로그인 금지
4. B/C에서 kFarmAI 자동작업 실행 금지
5. B/C에서 OpenClaw 설치 금지
6. B/C에서 배포 작업 금지
7. 회사 장비, 회사 네트워크, 회사 저장공간을 kFarmAI 개인 프로젝트 운영에 사용 금지

핵심 원칙:
개발은 A, 확인은 D, B/C 회사 자산은 제외한다.

2. 작업 전 반드시 지켜야 할 선행조건
================================================================

1. 현재 폴더 구조를 먼저 확인한다.
2. 작업 폴더가 C:\Users\user\PERSONAL\dev\kfarmai-web 인지 확인한다.
3. git status를 먼저 확인한다.
4. 어떤 파일을 수정할지 먼저 목록으로 정리한다.
5. 새로 만들 파일 목록을 먼저 정리한다.
6. 기존 파일을 바로 수정하지 않는다.
7. 수정 전 backup_YYYYMMDD_HHMM 폴더를 만들고 관련 파일을 백업한다.
8. 기존 기능을 깨뜨리지 않는 것을 최우선으로 한다.
9. 작업 범위를 임의로 넓히지 않는다.
10. 요청받은 기능 외의 리팩토링, 디자인 변경, 파일명 변경, 구조 변경은 하지 않는다.
11. 기존 index.html, agri_map.html, santo.html, seed.html, cpa.html, fert.html, news_sub.html 파일은 특히 주의한다.
12. API 키, 비밀번호, 토큰, 개인정보를 코드에 직접 넣지 않는다.
13. .env, secrets, logs, backup 폴더는 Git에 올리지 않는다.
14. 새 기능은 작은 단위로 추가한다.
15. 작업 후 변경 파일 목록을 반드시 보고한다.
16. 실행 방법과 되돌리는 방법을 README 또는 작업 로그에 남긴다.
17. 작업 완료 후 테스트 체크리스트를 작성한다.
18. 문제가 발생하면 무리하게 고치지 말고 원인과 수정 후보를 먼저 정리한다.
19. 실제 배포, Git push, 장시간 자동 실행은 사용자가 확인하기 전에는 하지 않는다.

3. kFarmAI 핵심 서비스 방향
================================================================

kFarmAI는 다음 5개 축으로 설계한다.

1. SEO 지식글
- 실제 농민과 홈가드닝 사용자가 검색하는 질문 기반
- 예: 벼 모판 검게 변함, 상토 입상 굳음, 농약 혼용 가능 여부, 딸기 고설 EC
- 공식기관 출처를 포함한 Q&A형 콘텐츠
- Google 검색 유입 확보 목적

2. AI 진단
- 사진 업로드 또는 증상 입력 기반
- 병해충, 생육장애, 상토 문제, 분갈이 문제 등을 진단 보조
- 단정적 처방 금지
- 원인 후보, 확인할 점, 조치 방향 중심

3. 공식자료 기반 답변
- 농촌진흥청, 농사로, 농약안전정보시스템, 농림축산식품부, 지자체 공고 등 우선
- 민간 블로그/카페는 참고 수준
- 농약·병해충·보조사업은 반드시 출처 표시

4. 지역 연결
- 주변 농약사, 농자재점, 종묘사, 상토/비료/농약 관련 업체, 지역 보조사업 공고

5. 수익화
- 농약사 지도 상단 노출, 농자재 업체 유료 등록, 상토/종자/비료/농약 업체 광고, AI 진단 고도화, 농업 검색 데이터 기반 B2B 리포트

================================================================
4. 절대 하면 안 되는 방향
================================================================

1. “농업 ChatGPT”처럼 범용 AI 검색으로 만드는 것
2. 농약명을 단정 추천하는 것
3. 사진만 보고 병명을 확정하는 것
4. 검수 없이 농약·병해충 글을 자동 공개 발행하는 것
5. 업체 디렉토리만 대량으로 만드는 것
6. 처음부터 커뮤니티 중심으로 만드는 것
7. 농자재 쇼핑몰처럼 보이게 만드는 것
8. 출처 없는 AI 답변을 대량 생성하는 것
9. 제목만 다르고 본문이 비슷한 저품질 SEO 글을 대량 공개하는 것
10. 사용자의 개인정보를 동의 없이 저장하는 것
11. API 키나 비밀번호를 Git에 올리는 것
12. 기존 파일 구조를 임의로 뒤엎는 것
13. 사용자가 확인하기 전 자동 배포하는 것
14. 회사 장비 B/C를 개인 개발에 사용하는 것

5. SEO 콘텐츠 개발 원칙
================================================================

SEO 글 구조:
1. 실제 사용자가 물어볼 만한 질문형 제목
2. 짧은 빠른 답변
3. 가능한 원인
4. 현장에서 확인할 점
5. 바로 할 수 있는 조치
6. 주의사항
7. 공식 출처
8. 관련 질문
9. AI 진단 또는 주변 농약사 연결 CTA

SEO 필수 요소:
- title
- meta description
- canonical
- og:title
- og:description
- og:url
- Article JSON-LD
- FAQPage JSON-LD
- sitemap.xml 반영
- robots.txt sitemap 경로 유지

주의:
- sitemap 제출은 색인을 보장하지 않는다.
- Google Indexing API는 일반 SEO 글에 사용하지 않는다.
- Search Console에는 sitemap 제출 방식으로 접근한다.

6. 농약·병해충 관련 안전 원칙
================================================================

농약, 약제, 살균제, 살충제, 제초제, 혼용, 희석배수, 병해충, 탄저병, 무름병, 바이러스, 방제, 처방 등의 키워드가 포함되면 반드시 주의문구를 넣는다.

필수 주의문구:
“농약 사용은 반드시 등록 농약 여부, 적용 작물, 적용 병해충, 희석배수, 안전사용기준을 확인해야 합니다. 현장 증상만으로 약제를 단정하기 어렵기 때문에 지역 농업기술센터, 농약사, 전문가 상담을 권장합니다.”

농약 관련 글에서 금지:
- 특정 농약명을 단정 추천
- 희석배수를 임의로 안내
- 병명을 확정
- 혼용 가능 여부를 단정
- 등록 여부 확인 없이 사용 안내
- 안전사용기준 생략

농약 관련 글에서 허용:
- 확인 방법 안내
- 농약안전정보시스템 링크 안내
- 전문가 상담 권장
- 적용 작물/병해충/안전사용기준 확인 필요성 설명

7. AI 진단 기능 설계 원칙
================================================================

AI 진단은 “확정 진단”이 아니라 “진단 보조”로 설계한다.

결과 화면 구조:
1. 가능 원인 후보
2. 추가로 확인할 증상
3. 위험도
4. 현장에서 바로 확인할 점
5. 조치 방향
6. 공식자료 링크
7. 주변 농약사/농업기술센터 상담 안내
8. 관련 Q&A 글 연결

표현 원칙:
- “~일 수 있습니다”
- “~가능성이 있습니다”
- “현장 확인이 필요합니다”
- “정확한 판단은 전문가 상담을 권장합니다”

금지 표현:
- “이 병입니다”
- “이 농약을 쓰세요”
- “무조건 이렇게 하세요”
- “확실합니다”

8. 커뮤니티 개발 원칙
================================================================

커뮤니티는 초기 핵심 기능이 아니다. 초기에는 검색 유입과 Q&A 지식글을 우선한다.

커뮤니티 확장 조건:
1. 월 방문자 5,000명 이상
2. 사용자 질문 로그 누적
3. AI 진단 클릭 발생
4. 농약사 지도 이동 발생
5. 반복 방문자 발생

초기 커뮤니티:
- 질문 남기기
- 증상 사진 공유
- 지역별 농업 질문
- 홈가드닝 자랑
- 해결 사례 모음

주의:
처음부터 빈 게시판을 많이 만들지 않는다.

9. 데이터 저장 원칙
================================================================

초기에는 간단한 JSON 또는 정적 파일로 시작해도 된다. 하지만 나중에 DB 전환이 가능하도록 구조를 명확히 한다.

저장할 수 있는 데이터:
- SEO 질문, 글 slug, 카테고리, 작물명, 출처 URL, 생성일, 수정일, 상태값 draft/published
- 사용자 검색어, 클릭 로그, AI 진단 이동 여부, 농약사 지도 클릭 여부

개인정보 원칙:
- 이름 저장 금지
- 전화번호 저장 금지
- 주소 저장 금지
- 원본 IP 저장 금지
- 필요한 경우 IP는 해시 처리
- 민감정보는 저장하지 않음
- 마케팅 수신은 동의 기반으로만

상태값:
- draft: 자동 생성 초안
- review: 검수 대기
- published: 공개 가능
- archived: 비공개 또는 보관

농약/병해충/보조사업 글은 기본적으로 draft 또는 review 상태로 생성한다.

공통:

- 모바일 화면 깨짐 없음
- 기존 메뉴 정상
- 기존 링크 정상
- 콘솔 에러 없음
- 파일 인코딩 UTF-8 유지

SEO:
- kb/index.html 열림
- 개별 kb 글 열림
- title 정상
- meta description 정상
- canonical 정상
- FAQ JSON-LD 정상
- Article JSON-LD 정상
- 출처 링크 정상
- sitemap.xml 정상
- robots.txt 정상

자동화:
- data 파일 생성 정상
- logs 기록 정상
- 중복 생성 방지
- 에러 발생 시 중단/로그 처리
- draft 상태 유지

농약/병해충:
- 주의문구 있음
- 단정적 처방 없음
- 공식 출처 있음
- 전문가 상담 권장 있음

16. kFarmAI 개발 우선순위
================================================================

1순위: SEO 지식글 자동 생성/검수/발행 구조
2순위: 공식 출처 기반 답변 구조
3순위: AI 진단 진입 버튼과 기본 화면
4순위: 농약사 지도 연결
5순위: 사용자 질문 로그 수집
6순위: Search Console 기반 키워드 분석
7순위: 지역/작물별 허브 페이지
8순위: 업체 상세 페이지
9순위: 광고/유료등록 테스트
10순위: 커뮤니티 확장

지금 당장 커뮤니티, 결제, 회원가입, 복잡한 관리자 기능부터 만들지 않는다.

17. 벤치마킹 방향
================================================================

kFarmAI는 다음 모델을 조합한다.

- Perplexity: 출처 기반 답변
- Plantix: 사진 진단
- AgroStar: 농업 상담과 농자재 연결
- Nextdoor: 지역 기반 연결
- Houzz: 콘텐츠에서 업체 연결
- AlphaSense: 데이터 리포트
- Klaviyo: 행동 데이터 기반 재방문/마케팅

현실적인 포지션:
“농업판 Perplexity + Plantix + 지역 농약사 연결 플랫폼”
