0. 핵심 결론
================================================================

1. OpenClaw는 kFarmAI 개발을 직접 대신하는 도구가 아니다.
2. OpenClaw는 여러 AI 도구와 로컬 작업을 연결하는 개발 지휘실/게이트웨이로 본다.
3. 실제 코드 수정은 A 개인 개발 PC에서만 한다.
4. B 회사 PC와 C 회사 핸드폰은 kFarmAI 개인 개발에 사용하지 않는다.
5. D 개인 핸드폰은 확인/모니터링/비상중지 용도로만 사용한다.
6. OpenClaw는 A 개인 PC에만 설치·운영한다.
7. MCP는 무조건 많이 연결하지 않는다.
8. kFarmAI에 필요한 최소 기능만 안전하게 연결한다.
9. 지금 당장 추천하는 MCP는 Claude ↔ Higgsfield 정도이다.
10. 파일시스템, GitHub, 터미널 MCP는 초기에는 보류한다.

2. OpenClaw의 개념
================================================================

OpenClaw는 여러 AI 모델과 채널, 로컬 작업을 연결하는 개발 지휘실이다.

역할 비유:
대표님 = 총괄 PM
OpenClaw = AI 작업 지휘실/게이트웨이
Codex/ChatGPT = 코드 작성·수정
Claude = 긴 문서 검토·리스크 분석·설계 정리
Gemini = 최신 자료 조사·공식자료 확인
Lovable = UI 프로토타입 제작
Higgsfield = 이미지/영상 생성
GitHub = 코드 저장소
A 개인 PC = 실제 작업 공간

OpenClaw는 작업을 정리하고, 모델별 역할을 나누고, 로그를 남기고, 필요한 경우 외부 도구를 연결하는 운영 도구이다.

11. MCP 설정 판단
================================================================

결론:
지금은 “필수 MCP만 최소로” 설정한다.

추천:
1. Claude ↔ Higgsfield MCP
- 추천
- 목적: 이미지/영상 생성

- 보류
- 이유: 자동 커밋/푸시 위험

4. 터미널/PowerShell 실행 MCP
- 비추천
- 이유: 파일 삭제, 자동 실행, 보안 사고 위험

5. 브라우저/웹검색 MCP
- 공식자료 검색용으로 나중에 검토
- 현재는 ChatGPT/Gemini 수동 검색이 더 안전

핵심:
MCP는 많이 연결할수록 편하지만 위험도 커진다. 초기에는 Higgsfield MCP 정도만 우선 연결한다.

================================================================

주의:
- 회사 장비 B/C에는 설정하지 않는다.
- 개인 계정에서만 연결한다.
- 크레딧 소모를 확인한다.
- 제품 로고/포대/상표 표현은 검수한다.
- 농약/병해충 이미지는 오해 없게 예시 이미지로 표시한다.
