kFarmAI 개발 전 필수 안전 프롬프트

작업 폴더:
C:\Users\user\PERSONAL\dev\kfarmai-web

장비 원칙:
A = 개인 개발 PC
D = 개인 핸드폰
B = 회사 PC
C = 회사 핸드폰

kFarmAI는 개인 프로젝트이다.
개발은 A 개인 PC에서만 수행한다.
D 개인 핸드폰은 모바일 확인/비상 모니터링 용도로만 사용한다.
B 회사 PC와 C 회사 핸드폰은 kFarmAI 개인 개발에 사용하지 않는다.

작업 전 반드시 지켜라:
1. 현재 폴더 구조를 확인한다.
2. 작업 폴더가 C:\Users\user\PERSONAL\dev\kfarmai-web 인지 확인한다.
3. git status를 확인한다.
4. 수정 예정 파일 목록을 먼저 출력한다.
5. 새로 만들 파일 목록을 먼저 출력한다.
6. 기존 파일 수정 전 backup_YYYYMMDD_HHMM 폴더에 백업한다.
7. 기존 index.html, agri_map.html, santo.html, seed.html, cpa.html, fert.html, news_sub.html을 깨뜨리지 않는다.
8. 작업 범위를 임의로 넓히지 않는다.
9. 요청하지 않은 리팩토링, 디자인 변경, 파일명 변경, 구조 변경은 하지 않는다.
10. API 키, 비밀번호, 토큰, 개인정보를 코드에 직접 넣지 않는다.
11. .env, secrets, logs, backup 폴더는 Git에 올리지 않는다.
12. 농약/병해충/보조사업 관련 내용은 단정하지 않는다.
13. 농약 관련 내용에는 등록 농약, 적용 작물, 적용 병해충, 희석배수, 안전사용기준 확인 필요성을 안내한다.
14. 자동 생성 글은 draft 또는 review 상태로 둔다.
15. 사용자가 허락하기 전 실제 배포, Git push, 장시간 자동 실행은 하지 않는다.
16. 회사 장비 B/C 접근, 저장, 실행, 동기화는 하지 않는다.

작업 시작 전 먼저 출력할 것:
1. 이해한 작업 목표
2. 수정 예정 파일
3. 새로 만들 파일
4. 백업 대상
5. 테스트 방법
6. 위험 요소

중요:
먼저 위 6가지를 출력하고, 내가 “진행해”라고 하기 전에는 파일을 수정하지 마라.

작업 완료 후 반드시 출력할 것:
1. 변경 파일 목록
2. 실행 방법

4. 되돌리는 방법
5. 문서 업데이트 여부
6. 남은 문제 또는 다음 작업
